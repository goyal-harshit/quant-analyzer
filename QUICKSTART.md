# QuantAI — Quick Start Guide (Windows dev environment)

Local development on Windows with automatic port management and batch-file launchers. For general project setup (Docker or manual), see [README.md](README.md).

## Get started

Prerequisite: [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running, and `backend/.env` created (`cp backend/.env.example backend/.env`).

```
Double-click: start-dev.bat
```

The script will:
1. Check that Docker and Docker Compose are installed
2. Run the port checker (`check-ports.ps1`) and resolve any conflicts
3. Start all services with `docker-compose up -d --build`
4. Show service status and open http://localhost:3000 in your browser

### Default ports

| Service | Port | Purpose |
|---------|------|---------|
| Frontend | 3000 | Next.js UI (http://localhost:3000) |
| Backend API | 8000 | FastAPI REST endpoints (http://localhost:8000/api/v1, docs at /docs) |
| PostgreSQL | 5432 | Primary database (TimescaleDB image) |
| Redis | 6379 | Cache / Celery broker |
| Ollama | 11434 | Local LLM inference |
| Celery worker + beat | — | Background data ingestion (no exposed port) |

## Launcher scripts

| Script | Purpose |
|--------|---------|
| `start-dev.bat` | Launch the entire stack (with port check) |
| `stop-dev.bat` | Stop all services (`docker-compose down`; data volumes are preserved) |
| `status-dev.bat` | Show running containers and current port assignments |
| `logs-dev.bat` | Interactive log viewer (pick a service, or all; Ctrl+C to exit) |

Full script reference: [DEV_SCRIPTS.md](DEV_SCRIPTS.md).

## Port management

`start-dev.bat` runs `check-ports.ps1` before starting services:

- If a target port is busy, the next free port is used instead.
- Assignments are saved to `.ports.json` so the same ports are reused next session.
- Host-port mappings are written to `docker-compose.override.yml`, which Docker Compose picks up automatically.

Both `.ports.json` and `docker-compose.override.yml` are **local to your machine and gitignored** — they are regenerated by the port checker, so don't edit `docker-compose.override.yml` by hand.

To change a port manually: edit the `target` value in `.ports.json`, then run

```
pwsh -NoProfile -ExecutionPolicy Bypass -File "check-ports.ps1"
```

and restart with `stop-dev.bat` / `start-dev.bat`.

Check current assignments any time with `status-dev.bat`.

## Troubleshooting

**Services won't start**
Run `logs-dev.bat` and select the failing service (1 = backend, 2 = frontend) to see the error. Common causes: `backend/.env` missing, or the database still initializing (wait ~30 s and check again).

**Port already in use**
Just run `start-dev.bat` again — the port checker detects the conflict and picks the next available port.

**Need to stop everything**
Run `stop-dev.bat`. Containers and networks are removed; database and Ollama volumes are preserved.

## Command-line equivalents

If you prefer the terminal:

```powershell
docker-compose up -d          # start services (skips the port check)
docker-compose ps             # status
docker-compose logs -f backend
docker-compose down           # stop
```

## Project structure (dev tooling)

```
project/
├── start-dev.bat               ← launch the stack
├── stop-dev.bat                ← stop services
├── status-dev.bat              ← check running services
├── logs-dev.bat                ← view service logs
├── check-ports.ps1             ← port availability checker
├── .ports.json                 ← port registry (auto-managed, gitignored)
├── docker-compose.yml          ← service definitions
├── docker-compose.override.yml ← port overrides (auto-generated, gitignored)
├── backend/                    ← FastAPI application
└── frontend/                   ← Next.js application
```
